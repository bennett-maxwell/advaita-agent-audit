# Learning Extraction Skill v1

## Purpose
After every significant task, AI extracts what was learned, what worked, what didn't, and adds to the knowledge base. FKI's collective intelligence grows with every execution.

## Learning Extraction Trigger
- After any multi-step skill execution
- After any error that was resolved (via ERRORS.md)
- After any human gate decision by Bennett
- After any work order completed by Leo/Ivan
- After any candidate deal closes or drops

## Learning Format (saved to Drive .learnings/ folder)
```
## LEARN-[YYYYMMDD]-[NNN]
Context: [what task was being done]
What worked: [specific approach that succeeded]
What failed: [specific approach that failed]
Root cause: [why it failed]
Fix: [what resolved it]
Generalizable insight: [1-sentence lesson that applies beyond this specific case]
Skill update recommended: [YES/NO — which skill + what change]
Memory update recommended: [YES/NO — what memory to add or update]
```

## Weekly Learning Digest
All learnings from the week compiled and posted to #leo-auto Sundays.
Any skill update recommendations → feed into weekly skill audit.

## Autonomy Value
**Replaces:** Institutional knowledge disappearing when sessions end or agents change.
**Result:** Every mistake teaches the system. Every success gets codified. FKI's AI gets measurably smarter every week.
**Advaita gap closed:** Institutional learning — from 0% → 90% systematic.
